What is this?

The Royal Game of Ur is one of the oldest board games we know of — people were playing it in Mesopotamia around 2600 BCE. Its rules were brilliantly reconstructed from a cuneiform tablet by Dr Irving Finkel at the British Museum. I wanted to play it the way I most enjoy old games: on old machines. So I built it for a whole shelf of them.

One game, eleven machines

The trick to keeping eleven ports sane is a single shared C core — the rules, the AI, the dice, the network protocol — that compiles for two completely different worlds: the 6502 machines (Atari, C64, Apple II, NES…) via cc65, and the Z80-family ones (Coleco Adam, Master System, Game Boy…) via z88dk. Same game brain, wildly different bodies. On top of that sits a thin per-machine layer for graphics, sound and input.

The current lineup: Atari 8-bit, Atari 5200, Commodore 64, Apple IIe, Coleco Adam, ColecoVision, Sega Master System, Game Gear, Game Boy / Color, and NES/Famicom.

Making each one look right

Every port draws the same authentic Standard of Ur board — carved cells, gold flower rosettes, bullseye eyes, two-tone tokens — but rendered to each machine's strengths: hardware sprites here, redefined character sets there, raw bitmap on the Apple II (which gets both a 16-colour lo-res board and a double-hi-res one). Getting the same look out of a VIC-II, a TMS9918, GTIA and the Game Boy's DMG was half the fun.

The oldest melody, on the chips I had

The title music is the Hurrian Hymn (Ugarit, c.1400 BCE) — the oldest substantially-complete written melody we have. It plays on POKEY, SID, the SN76489, the NES & Game Boy sound chips, and the Apple II's 1-bit speaker. That last one had a great little war story: the speaker was dead silent for ages because the C compiler was quietly optimising away the very instruction that clicks the speaker cone. One line of inline assembly later, it sings — and if you've got a Mockingboard, the Apple II build will find it and use the AY chips instead.

…and they can play each other

Because the network protocol lives in that shared core, the ports speak the same language over the wire. Through FujiNet, two machines can play a real networked game mediated by a little game server. I've had two Ataris playing each other end-to-end through emulated FujiNet, and the Coleco Adam build speaks the same protocol. A Game-Boy-vs-C64 match is the whole reason the core is built this way — that's where this is headed.

Where you come in

This is very much a living project, and real-world testing beats anything I can do solo. I'd love to hear:

  • Does it run on your hardware? Real machines especially — which one, and did anything misbehave?
  • Bugs / weirdness — a wrong move, a graphics glitch, a sound that's off, a control that feels wrong.
  • What would make it better? Features, options, a machine I've missed.

Drop a comment here on itch, or open an issue on GitHub: github.com/slaapliedje/ur (all the source, every platform, and the server live there — it's free and GPLv3).

Thanks for taking a look. May the rosettes favour you. 🌺
