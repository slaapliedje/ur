Three releases have landed since the last post (v0.1.3, v0.1.4, v0.1.5), and they add up to the biggest update this project has had: the 16-bit era is here, every port got smarter about showing you your moves, online play is now zero-setup, and the whole source code is public. Let's go oldest-city-first.

The ziggurat rises: Atari ST, STe, TT & Falcon

The shared game brain that runs every 8-bit port made its third leap — after the 6502 and the Z80 it now compiles for the 68000 — and the result is four new Atari downloads from one platform layer:

  • Atari ST — 320×200 in 16 colours from 512.
  • Atari STe — the same board, but every colour re-picked from the STe's 4096-colour palette. Subtle everywhere, obvious at sunset.
  • Atari TT — 256 colours on screen: a real gradient sky, and cell faces lit top-to-bottom the way only the Falcon could do before.
  • Atari Falcon — the flagship: true-colour RGB565, gradient everything.

They all open on a new title screen: the Great Ziggurat of Ur at dusk — stepped mud-brick terraces, the grand staircase, a setting sun, and the shrine on top glazed blue, because the accounts of the real ziggurat at Ur say the temple crowning it was faced in blue-glazed brick. Like all the art in this project it's drawn procedurally — there isn't a single image file in the ROMs; the ziggurat is maths.

The board got a dimension too: the whole playfield now sits on the screen like a raised stone slab — a drop shadow under the board's silhouette, crisper carved bevels, and pieces that cast their own little shadows with a specular glint on top.

Two new download buttons on the page: Atari ST (the ST, STe and TT programs together) and Atari Falcon. All four are GEMDOS programs — drop one on a drive and run it, on real hardware or in Hatari (EmuTOS works fine for the ST/STe/TT; the Falcon wants TOS 4).

See the play before you make it

Every port now lights up the square you'd land on while you're choosing a move — and on the tile-based machines it's done as a tinted cell: the square turns green with its rosette or mosaic still visible, so the board stays readable. One fun constraint: the Game Boy's four colours don't include a green, so on the GB the destination cell glows gold instead — the palette swap keeps the motif legible in both colour and original grey-brick DMG mode.

Alongside that, the move list flags captures everywhere (no more losing a piece to a surprise), and the consoles got a proper difficulty menu — a little gold rosette cursor you move with the D-pad — instead of the old “Up for Easy, Down for Hard” guesswork.

The case of the garbled C64 board

A fix worth confessing: the Commodore 64 build had started drawing its board as garbage glyphs, and the cause is peak 8-bit. The program loads at $0801 and grows upward; the custom character set lived at $3800; and the game had quietly grown to the point where the font was being copied over the program's own running code. A 13K program and a 2K charset simply don't fit under $4000 together. The C64 now renders with the ROM charset plus its hardware-sprite tokens (the same renderer the FujiNet build uses) — clean board, no more memory Tetris. If your C64 copy ever looked scrambled: grab the new one.

Online is now download-and-go

The FujiNet side took two steps this cycle. First, the C64 and Apple II each gained a dedicated online build (ur-online in their downloads) speaking the same wire protocol as the Atari and Adam — so all four machines can meet at the same table. Second, as of v0.1.5 every online build points at the live server out of the box. Pick “Online” and you're connecting — no host to type, nothing to configure. (It's still changeable in-game, and the FujiNet Lobby handoff still overrides it, but the default now Just Works.)

The source is public

As of this week the whole project is open: one portable C core (rules, AI, protocol) driving thin per-machine layers across three CPU families — 6502, Z80, and now 68000 — and thirteen downloads. GPLv3, warts and build scripts and all: github.com/slaapliedje/ur. If you're curious how the same game fits in an NES cart, an Apple II disk, and a Falcon program, it's all there.

What's next

The 68000 door is open, so: Mega Drive / Genesis and Amiga are next in line, plus some ST-family polish (STe DMA sound, the blitter). And as ever — if you run any of these on real hardware, especially the new ST/STe/TT/Falcon builds, I'd love to hear how they behave. The best fixes in this project have come from real CRTs and real machines catching what emulators politely hide.
